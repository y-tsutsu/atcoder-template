# 典型考察メモ

問題を解く方針に迷ったとき、条件や制約から使えそうな考え方を探すためのメモ。

## 構築・復元

### 操作を逆順に考える

- 分割、削除、合成を繰り返した結果だけが与えられる問題では、最後の操作から逆に戻せないか考える。
- 最後に操作された要素には、端にある、全体の一辺と長さが一致するなど、見つけやすい特徴が残ることがある。
- 解の存在が保証されているなら、「条件を満たす要素が必ず一つは存在する」と考えて貪欲に取り除けないか試す。
- 要素を一つ取り除いた後も同じ形の問題が残るなら、帰納的に処理できる可能性が高い。

例：長方形を直線で切り分けてできたピースを復元する場合、残り領域の高さ全体または幅全体を占めるピースを端から取り除く。

### 二次元平面を蛇行する順に並べる

- 二次元上の点を近い順に巡回したい場合、片方の座標をブロック分割し、ブロックごとにもう片方の座標を昇順・降順へ交互に並べる。
- 同じ向きへ毎回戻るのではなく蛇行させることで、ブロック内における移動量を座標幅程度に抑えられる。
- 点の個数を `N`、座標幅を `W`、ブロック幅を `B` とすると、移動量を `W^2/B + NB + O(W)` 程度に評価できる。
- `B = W/sqrt(N)` 程度にすると、主要部分の移動量が `O(W sqrt(N))` になる。
- この順序はMo's algorithmで区間クエリを並べる方法と同じで、幾何の構築にも流用できる。
- 巡回路では開始位置をいったん無視して閉路を作り、最後に列をrotateして指定された頂点を先頭にできる。

## 数え上げ・シミュレーション

### 二次元の数え上げを一次元全探索へ落とす

- 二次元の全点を走査できなくても、片方の座標だけ全探索し、もう片方を条件を満たす区間の長さとして `O(1)` で数えられないか考える。
- 閉形式や二次元累積和を作る前に、制約から実際のループ回数を計算する。数百万回なら単純な全探索も候補に入れる。
- `max(a, b)` が条件に現れたら、`a > b` と `a <= b` のように、どちらが最大になるかで重複なく分割する。
- 絶対値を含む場合も、`|x| > |y|` なら `-|x| < y < |x|` のように、固定した座標に対する区間へ変形する。

例：`max(|x|, |y|)` が偶数となる点は、`|x| > |y|` では偶数の `x`、`|x| <= |y|` では偶数の `y` を全探索して数える。

### 出現回数の分布をレベルごとに更新する

- 操作が各値の出現回数だけで決まるなら、長くなる列そのものではなく、値ごとの出現回数を見る。
- 最小の出現回数を持つ値を増やす操作では、同じ出現回数の値をグループ化し、次の出現回数までまとめて引き上げられないか考える。
- `values_by_count[c]` のように、出現回数 `c` の値をまとめておくと、レベルを上げながら候補集合へ追加できる。
- タイブレークが値の昇順なら、候補集合を昇順に一周した列が、一段分の追加列になる。
- 初期列の長さが `N` なら初期出現回数は最大 `N` なので、回数のレベルは高々 `0...N` まで調べればよい。
- 全種類の出現回数が等しくなった後は、全種類をタイブレーク順に並べた周期列になることがある。
- 非常に先の位置を問われる場合は、有限の過渡部分を処理した後、周期部分を剰余で答える。
- 追加列の途中にあるクエリへ答えるには、候補集合の `k` 番目を取得できる順序付き集合やBITなどを使う。

## 数論

### 巨大な整数の商をmodで求める

- 巨大な整数 `N` に対して `floor(N/M) mod P` が欲しい場合、`N` そのものを復元する必要はない。
- `r = N mod (MP)` を求めれば、答えは `floor(r/M)` になる。
- `N = Mq + s` とおくと、`N mod (MP) = M(q mod P) + s` になることから確認できる。
- 桁列が圧縮されている場合も、必要な法のもとで連結した値だけを更新する。

### 割り算を含む値のmod

- `(A/D) mod M` を求めたいとき、`D` が `A` を割り切るなら、`A mod (DM)` を求めてから通常の整数として `D` で割れる。
- `D` のmod逆元が存在しない場合でも使える。
- レピュニットでは `(10^n - 1)/9` を直接mod上で9の逆元に掛けず、`10^n - 1 mod (9M)` を9で割る。

### 剰余を円周上の点として扱う

- 値に同じ数 `K` を何度でも加減できる場合、`K` で割った余りが本質になる。
- `0...K-1` の余りは端がつながった円周上の点と考え、どこを切って直線にするかを試す。
- 余りをソートし、先頭を取り出して `K` を加えて末尾へ移すと、円周の切れ目を順番に変更できる。
- 各切れ目における最大値と最小値だけで評価できるなら、dequeや配列を2倍した尺取りで全候補を調べられる。

### 一つだけ除いた結果をすべて求める

- 各要素を一つずつ除外した答えが必要でも、毎回答えを最初から計算し直さない。
- 演算に逆演算があれば、全体の結果から対象要素の寄与を取り除く。
- `max`、`min`、LCMなど簡単に寄与を取り除けない演算では、最大値と2番目の最大値、および最大値の個数を持つ方法を考える。
- 素因数分解を使うLCMでは、素数ごとに指数の最大値と2番目の最大値を管理する。

### 多数の整数を素因数分解する

- 一つの整数だけなら `O(sqrt(N))` の試し割りで十分なことが多い。
- 値の上限が小さく、素因数分解する個数が多い場合は、最小素因数を篩で前計算する。
- 「値の上限」と「クエリ数」の積が大きいときは、一回の処理が軽く見えても試し割りでは間に合わない可能性を疑う。

## 木DP

### 親へ返す値の意味を決める

- まず適当な頂点を根にして、各辺を親子関係として見る。
- `dp[v]` を式より先に日本語で定義する。「頂点 `v` の部分木における答え」だけではなく、端点、選択状態、親との接続方法まで明確にする。
- 親へ返す値は、親から辺を一本つないだ後も条件を満たす状態でなければならない。
- 親との辺を追加すると、現在の頂点の次数や立場が変わることに注意する。
- 葉など最小の部分木で `dp` が何になるかを先に決め、子の結果から同じ定義を保ったまま遷移できるか確認する。

### 親へ伸ばすDPと完成した答えを分ける

- 親へ返せる候補は通常一方向だけだが、現在の頂点で完成する答えには複数の子を使えることがある。
- 木上のパスを求める場合は、「子一つから伸ばして親へ返す値」と「子二つを現在の頂点で結合した答え」を分けて考える。
- 子二つを結合するなら、子から来た値の最大2個だけでよいことが多い。
- `dp[v]` だけでなく、各頂点を最高点や折り返し地点とした答えで、全体の答えを更新する。

### 端点と内部で条件が違うパス

- パスの端点と内部頂点で次数などの条件が違う場合、それぞれを混同しない。
- 子から伸ばしたパスを親へ返すと、現在の頂点は端点ではなく内部頂点になる。その状態で内部頂点の条件を満たすか確認する。
- 子一つと現在の頂点でパスを完成させる場合、現在の頂点は端点の条件を満たせばよい。
- 子二つを現在の頂点で結合する場合、現在の頂点は内部頂点の条件を満たす必要がある。
- 条件が元の木における次数なら、子の個数ではなく根を付ける前の `degree[v]` を使う。

例：背骨の各頂点に葉を2本付けるパスでは、背骨の端点は次数3以上、内部頂点は次数4以上必要になる。

## 再帰的な列・文字列

### 巨大な連結列をprefix集約で辿る

- 再帰的な連結で作られる巨大な列や文字列は、実体を構築せず、各ブロックの長さと必要な集約値だけを前計算する。
- `S_i = S_{i-1} + S_{i-2}` なら、長さと文字数など加算可能な情報も同じ規則で合成できる。
- 区間 `[L, R]` の集約値は、`prefix(R) - prefix(L-1)` のようにprefix関数2回へ変換する。
- prefixが左側ブロックに収まるなら左側へ降り、越えるなら「左側全体の集約値＋右側の残りのprefix」に分解する。
- 列の長さが指数的に増える場合、位置が `10^18` などでも必要な階層数は `O(log R)` 程度になる。
- 求める最大位置をprefixとして含む段階まで構築すれば、それより先の巨大な添字の列を直接扱う必要はない。
- 文字数に限らず、和、種類別個数、ハッシュなど、結合できる情報へ応用できないか考える。

## グラフ・行列累乗

### 候補を構築してから検証する

- 条件を直接判定しにくい場合、条件を満たすならこうなるはずという候補を構築し、最後に入力の全条件と一致するか確認する。
- 構築した候補が必要条件から得られたものでも、十分条件を満たすとは限らないため、検証までを解法に含める。
- 全頂点間距離が木の距離か判定する場合、距離を完全グラフの辺重みとみなし、MSTを木の候補として構築できることがある。
- MSTを構築できても元の距離行列を再現するとは限らないため、候補の木における全頂点間距離を最後に照合する。

### 密グラフのMSTはPrim法を考える

- 隣接行列が与えられた密グラフでは、全辺を列挙してソートするKruskal法より、隣接行列版Prim法を検討する。
- 隣接行列版Prim法は、現在の木へ最小コストで接続できる頂点を一つずつ追加し、`O(N^2)` 時間、`O(N)` 追加メモリでMSTを構築できる。
- 完全グラフでは辺が `O(N^2)` 本あるため、辺リストを持たないことでPythonのメモリ使用量を大きく抑えられる。

### 木上距離の求め方をクエリ数で選ぶ

- 根からの累積距離があれば、`dist(u, v) = dist[u] + dist[v] - 2 * dist[lca(u, v)]` で木上距離を求められる。
- ダブリングによるLCAは一回 `O(log N)` なので、全体では「一回の計算量×クエリ数」まで見積もる。
- 距離クエリが少数ならLCAが便利だが、全頂点間距離が必要なら、各頂点からDFSして合計 `O(N^2)` とする方が単純で速い場合がある。

### 無限過程を有限状態へ落とす

- 無限に続く漸化式や操作でも、次の状態を決めるために必要な情報が有限なら、有限グラフ上の遷移として扱える。
- 直前 `d` 項から次項が決まる漸化式では、直前 `d` 項を組にして一つの状態と考える。
- 倍数、余り、周期性だけが問題なら、値そのものではなく `mod M` の値を状態にできないか考える。
- 状態数と各状態からの遷移数を見積もり、全状態をメモ化またはグラフ探索できるか確認する。
- 各状態からの遷移先が一つならFunctional Graphになり、遷移は既知の状態かサイクルへ入る。
- 特定の状態へ到達するかを全状態について判定する場合は、メモ化探索だけでなく、辺を逆向きにして目的の状態からまとめて探索する方法も考える。

例：直前2項で次項が決まる漸化式を `mod M` で考える場合、連続する2項 `(x, y)` を状態にすると状態数は高々 `M^2` になる。

### 全prefixを差分更新する

- `k = 1, 2, ..., N` の各prefixについて答える問題では、各問題を最初から解き直さず、`k` を一つ増やした差分だけ処理できないか考える。
- 頂点、辺、使用可能な要素などが単調に追加されるなら、追加されたものを起点に状態を更新する。
- グラフの到達可能性では、新しく到達可能になった頂点からだけ探索を再開すると、各頂点・辺を全体で定数回しか処理せずに済むことがある。
- DFSやBFSを何度も呼んでいても、各要素を一度しか訪問しないなら、合計計算量は `O(N + M)` などに抑えられる。
- prefix外へ出る最初の要素だけを削除すればよい場合は、到達済み領域の境界を集合やカウンタで管理する。

### ちょうど K 回の遷移

- 「ちょうど `K` 回遷移する」「`K` が非常に大きい」「状態数は比較的小さい」なら、遷移の合成と繰り返し二乗法を考える。
- 通常の行列累乗だけでなく、求めるものに合わせて行列積の演算を置き換えられないか考える。
- `K` 回後の経路数は通常の加算・乗算、到達可能性は OR・AND、最小コストは min・加算で遷移を合成できる。

### min-plus 行列積

- `A[i][j]` を「頂点 `i` から頂点 `j` へ決められた回数で移動する最小コスト」とする。
- `a` 回の遷移と `b` 回の遷移は、`C[i][j] = min_k(A[i][k] + B[k][j])` で合成できる。
- 行列を繰り返し二乗法で `K` 乗すると、ちょうど `K` 回遷移した最小コストを `O(N^3 log K)` で求められる。
- 単位行列は、対角成分が `0`、それ以外が `INF`。
- 始点へ戻る最小コストが欲しい場合は、累乗結果の対角成分を見る。
